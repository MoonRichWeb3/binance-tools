## 1. 目标

本文用于设计一个类似 TradingView 的指标查看工具。核心目标是：

1. 明确前端图表和后端指标计算需要的数据格式。
2. 明确每类指标依赖哪些字段。
3. 明确常见指标的计算方式和输出结构。
4. 为后续实现指标计算模块、API 返回结构和图表渲染打基础。

本文只覆盖常见技术指标，不覆盖画线工具、策略回测、订单执行和复杂社区脚本系统。

## 2. 基础数据模型

### 2.1 K 线数据 Candle

所有价格类指标的基础输入都是 K 线。建议使用毫秒时间戳，并保证同一个交易对、同一个周期内时间连续且升序排列。

```json
{
  "exchange": "binance",
  "symbol": "BTCUSDT",
  "market_type": "spot",
  "interval": "1m",
  "open_time": 1715750400000,
  "close_time": 1715750459999,
  "open": "65000.00",
  "high": "65120.50",
  "low": "64980.00",
  "close": "65080.10",
  "volume": "123.456",
  "quote_volume": "8034567.89",
  "trade_count": 1280,
  "taker_buy_base_volume": "66.123",
  "taker_buy_quote_volume": "4300000.12"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| exchange | string | 是 | 交易所，例如 `binance`。 |
| symbol | string | 是 | 交易对，例如 `BTCUSDT`。 |
| market_type | string | 是 | 市场类型，例如 `spot`、`usdt_futures`。 |
| interval | string | 是 | K 线周期，例如 `1m`、`5m`、`1h`、`1d`。 |
| open_time | integer | 是 | K 线开始时间，毫秒时间戳。 |
| close_time | integer | 是 | K 线结束时间，毫秒时间戳。 |
| open | decimal string | 是 | 开盘价。 |
| high | decimal string | 是 | 最高价。 |
| low | decimal string | 是 | 最低价。 |
| close | decimal string | 是 | 收盘价。 |
| volume | decimal string | 是 | 基础资产成交量。 |
| quote_volume | decimal string | 建议 | 计价资产成交额。 |
| trade_count | integer | 可选 | 成交笔数。 |
| taker_buy_base_volume | decimal string | 可选 | 主动买入基础资产量，可用于 CVD 近似。 |
| taker_buy_quote_volume | decimal string | 可选 | 主动买入计价资产量。 |

价格和成交量建议在 API 中用字符串传输，避免浮点精度问题；计算层可转换为 Decimal 或高精度浮点。

### 2.2 衍生品扩展数据

加密货币永续合约常用指标还需要资金费率、持仓量、多空比等数据。这些数据不属于普通 K 线，应单独存储。

```json
{
  "exchange": "binance",
  "symbol": "BTCUSDT",
  "market_type": "usdt_futures",
  "timestamp": 1715750400000,
  "funding_rate": "0.0001",
  "open_interest": "82345.67",
  "open_interest_value": "5350000000.12",
  "long_short_ratio": "1.23"
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| funding_rate | decimal string | 可选 | 资金费率。 |
| open_interest | decimal string | 可选 | 未平仓合约数量。 |
| open_interest_value | decimal string | 可选 | 未平仓名义价值。 |
| long_short_ratio | decimal string | 可选 | 多空比，口径取决于交易所。 |

### 2.3 指标请求格式

建议指标计算 API 使用统一请求结构。

```json
{
  "exchange": "binance",
  "symbol": "BTCUSDT",
  "market_type": "spot",
  "interval": "5m",
  "limit": 500,
  "indicators": [
    {
      "name": "ema",
      "params": {
        "period": 20,
        "source": "close"
      }
    },
    {
      "name": "macd",
      "params": {
        "fast": 12,
        "slow": 26,
        "signal": 9,
        "source": "close"
      }
    }
  ]
}
```

### 2.4 指标输出格式

输出建议按时间对齐，便于前端直接叠加到 K 线图或副图。

```json
{
  "symbol": "BTCUSDT",
  "interval": "5m",
  "indicators": {
    "ema_20": [
      {
        "time": 1715750400000,
        "value": "65012.34"
      }
    ],
    "macd_12_26_9": [
      {
        "time": 1715750400000,
        "macd": "12.34",
        "signal": "10.11",
        "histogram": "2.23"
      }
    ]
  }
}
```

实现约定：

1. 指标未达到最小周期前，返回 `null` 或不返回该点，二选一并保持全局一致。
2. 所有输出点必须带 `time`，与 K 线 `open_time` 对齐。
3. 多线指标要明确每条线的名字，例如 `upper`、`middle`、`lower`。
4. 前端只负责展示，不应重复实现复杂计算逻辑。

## 4. 通用价格源

很多指标允许选择 source。建议先支持以下价格源：

| source | 计算方式 |
| --- | --- |
| open | `open` |
| high | `high` |
| low | `low` |
| close | `close` |
| hl2 | `(high + low) / 2` |
| hlc3 | `(high + low + close) / 3` |
| ohlc4 | `(open + high + low + close) / 4` |

V1 默认使用 `close`。VWAP、CCI、MFI 常用 `hlc3` 作为典型价。

## 5. 通用计算函数

### 5.1 SMA

```text
SMA[i] = sum(source[i - period + 1 ... i]) / period
```

最少需要 `period` 个数据点。

### 5.2 EMA

```text
alpha = 2 / (period + 1)
EMA[i] = source[i] * alpha + EMA[i - 1] * (1 - alpha)
```

首个 EMA 可用前 `period` 个数据的 SMA 初始化。

### 5.3 RMA / Wilder 平滑

RSI、ATR、ADX 常使用 Wilder 平滑。

```text
RMA[i] = (RMA[i - 1] * (period - 1) + value[i]) / period
```

首个 RMA 可用前 `period` 个值的 SMA 初始化。

### 5.4 标准差

```text
mean = SMA(source, period)
variance = sum((source - mean)^2, period) / period
stddev = sqrt(variance)
```

如需更贴近统计样本标准差，可用 `period - 1` 作分母；项目内必须固定一种口径。

### 5.5 True Range

```text
TR[i] = max(
  high[i] - low[i],
  abs(high[i] - close[i - 1]),
  abs(low[i] - close[i - 1])
)
```

第一根 K 线没有前收盘价时，可用 `high - low`。

## 6. 常见指标计算方式

### 6.1 SMA / MA

输入：`source`、`period`

```text
value = SMA(source, period)
```

输出：

```json
{
  "time": 1715750400000,
  "value": "65012.34"
}
```

### 6.2 EMA

输入：`source`、`period`

```text
value = EMA(source, period)
```

输出同 SMA。

### 6.3 MACD

输入：`source`、`fast`、`slow`、`signal`

```text
fast_ema = EMA(source, fast)
slow_ema = EMA(source, slow)
macd = fast_ema - slow_ema
signal_line = EMA(macd, signal)
histogram = macd - signal_line
```

输出：

```json
{
  "time": 1715750400000,
  "macd": "12.34",
  "signal": "10.11",
  "histogram": "2.23"
}
```

### 6.4 RSI

输入：`source`、`period`

```text
change = source[i] - source[i - 1]
gain = max(change, 0)
loss = max(-change, 0)
avg_gain = RMA(gain, period)
avg_loss = RMA(loss, period)
rs = avg_gain / avg_loss
rsi = 100 - 100 / (1 + rs)
```

边界处理：

1. `avg_loss = 0` 且 `avg_gain > 0` 时，RSI 可记为 `100`。
2. `avg_gain = 0` 且 `avg_loss > 0` 时，RSI 可记为 `0`。
3. 两者都为 `0` 时，可记为 `50` 或沿用上一值，项目内固定即可。

### 6.5 Bollinger Bands

输入：`source`、`period`、`multiplier`

```text
middle = SMA(source, period)
dev = StdDev(source, period)
upper = middle + multiplier * dev
lower = middle - multiplier * dev
bandwidth = (upper - lower) / middle
percent_b = (source - lower) / (upper - lower)
```

输出：

```json
{
  "time": 1715750400000,
  "upper": "66000.00",
  "middle": "65000.00",
  "lower": "64000.00",
  "bandwidth": "0.0307",
  "percent_b": "0.73"
}
```

### 6.6 ATR

输入：`high`、`low`、`close`、`period`

```text
tr = TrueRange(high, low, close)
atr = RMA(tr, period)
```

ATR 通常不直接给买卖信号，更适合用于止损距离、波动过滤和 Supertrend。

### 6.7 Supertrend

输入：`high`、`low`、`close`、`period`、`multiplier`

```text
hl2 = (high + low) / 2
atr = ATR(period)
basic_upper = hl2 + multiplier * atr
basic_lower = hl2 - multiplier * atr
```

最终上下轨需要按前一根 K 线递推：

```text
final_upper =
  if basic_upper < final_upper_prev or close_prev > final_upper_prev
  then basic_upper
  else final_upper_prev

final_lower =
  if basic_lower > final_lower_prev or close_prev < final_lower_prev
  then basic_lower
  else final_lower_prev
```

趋势方向：

```text
if close > final_upper_prev => trend = up
if close < final_lower_prev => trend = down
otherwise trend = trend_prev
```

输出：

```json
{
  "time": 1715750400000,
  "value": "64200.50",
  "trend": "up"
}
```

### 6.8 ADX / DMI

输入：`high`、`low`、`close`、`period`

```text
up_move = high[i] - high[i - 1]
down_move = low[i - 1] - low[i]

+DM = if up_move > down_move and up_move > 0 then up_move else 0
-DM = if down_move > up_move and down_move > 0 then down_move else 0

TR = TrueRange(high, low, close)
+DI = 100 * RMA(+DM, period) / RMA(TR, period)
-DI = 100 * RMA(-DM, period) / RMA(TR, period)
DX = 100 * abs(+DI - -DI) / (+DI + -DI)
ADX = RMA(DX, period)
```

输出：

```json
{
  "time": 1715750400000,
  "plus_di": "28.30",
  "minus_di": "16.20",
  "adx": "24.60"
}
```

### 6.9 Stochastic

输入：`high`、`low`、`close`、`k_period`、`k_smooth`、`d_period`

```text
raw_k = (close - lowest(low, k_period)) / (highest(high, k_period) - lowest(low, k_period)) * 100
k = SMA(raw_k, k_smooth)
d = SMA(k, d_period)
```

### 6.10 CCI

输入：`high`、`low`、`close`、`period`

```text
tp = (high + low + close) / 3
ma = SMA(tp, period)
mean_deviation = SMA(abs(tp - ma), period)
cci = (tp - ma) / (0.015 * mean_deviation)
```

### 6.11 VWAP

输入：`high`、`low`、`close`、`volume`、`anchor`

```text
typical_price = (high + low + close) / 3
vwap = cumulative_sum(typical_price * volume, anchor) / cumulative_sum(volume, anchor)
```

`anchor` 可以是日内开始、周开始、月开始，或用户手动选择的时间点。

### 6.12 OBV

输入：`close`、`volume`

```text
if close[i] > close[i - 1] => obv[i] = obv[i - 1] + volume[i]
if close[i] < close[i - 1] => obv[i] = obv[i - 1] - volume[i]
otherwise => obv[i] = obv[i - 1]
```

### 6.13 MFI

输入：`high`、`low`、`close`、`volume`、`period`

```text
tp = (high + low + close) / 3
money_flow = tp * volume
positive_flow = money_flow if tp > tp_prev else 0
negative_flow = money_flow if tp < tp_prev else 0
money_ratio = sum(positive_flow, period) / sum(negative_flow, period)
mfi = 100 - 100 / (1 + money_ratio)
```

### 6.14 CMF

输入：`high`、`low`、`close`、`volume`、`period`

```text
multiplier = ((close - low) - (high - close)) / (high - low)
money_flow_volume = multiplier * volume
cmf = sum(money_flow_volume, period) / sum(volume, period)
```

当 `high = low` 时，`multiplier` 可记为 `0`。

### 6.15 Donchian Channels

输入：`high`、`low`、`period`

```text
upper = highest(high, period)
lower = lowest(low, period)
middle = (upper + lower) / 2
```

### 6.16 Pivot Points

输入：上一周期 `high`、`low`、`close`

```text
p = (high + low + close) / 3
r1 = 2 * p - low
s1 = 2 * p - high
r2 = p + (high - low)
s2 = p - (high - low)
r3 = high + 2 * (p - low)
s3 = low - 2 * (high - p)
```

日内图常用上一日数据，日线图可用上一周或上一月数据。

## 7. 加密货币衍生指标计算方式

### 7.1 Funding Rate

输入：交易所资金费率数据。

输出建议：

```json
{
  "time": 1715750400000,
  "funding_rate": "0.0001",
  "annualized_rate": "0.1095"
}
```

若资金费率每 8 小时结算一次，粗略年化：

```text
annualized_rate = funding_rate * 3 * 365
```

不同交易所结算频率不同，必须按交易所实际规则处理。

### 7.2 Open Interest

输入：`open_interest`、可选 `open_interest_value`

```text
open_interest_change = open_interest[i] - open_interest[i - 1]
open_interest_change_pct = open_interest_change / open_interest[i - 1] * 100
```

输出可包含当前值、变化值、变化百分比。

### 7.3 Long / Short Ratio

输入：`long_short_ratio`

```text
ratio = long_positions_or_accounts / short_positions_or_accounts
```

注意：交易所可能按账户数、持仓量或大户账户统计，不能混用。

### 7.4 CVD

优先输入：逐笔成交数据 `trades`，包含成交方向和成交量。

```text
delta = buy_volume - sell_volume
cvd = cumulative_sum(delta)
```

如果只有 Binance K 线里的 `taker_buy_base_volume`，可做近似：

```text
buy_volume = taker_buy_base_volume
sell_volume = volume - taker_buy_base_volume
delta = buy_volume - sell_volume
cvd = cumulative_sum(delta)
```

这是近似口径，不等于逐笔成交级别的精确 CVD。

## 8. 前端展示建议

主图指标：

| 指标 | 展示方式 |
| --- | --- |
| SMA / EMA | K 线图上叠加线。 |
| Bollinger Bands | K 线图上叠加上中下轨，可填充上下轨区域。 |
| Supertrend | K 线图上叠加趋势线，上涨和下跌用不同颜色。 |
| VWAP | K 线图上叠加单线。 |
| Donchian Channels | K 线图上叠加通道。 |
| Pivot Points | K 线图上叠加水平线。 |

副图指标：

| 指标 | 展示方式 |
| --- | --- |
| Volume | 柱状图，可按涨跌 K 线着色。 |
| MACD | MACD 和 Signal 用线，Histogram 用柱。 |
| RSI | 单线，建议显示 30/50/70 阈值线。 |
| ADX / DMI | ADX、+DI、-DI 三条线。 |
| Stochastic | %K、%D 两条线，显示 20/80 阈值线。 |
| OBV / MFI / CMF | 副图折线或柱线。 |
| Funding / OI / CVD | 副图折线或柱状变化。 |

## 9. V1 最小实现范围

建议第一版只做这组指标，既够用，也容易验证：

| 阶段 | 指标 | 原因 |
| --- | --- | --- |
| V1.0 | SMA、EMA、Volume | 最基础，验证 K 线和叠加线能力。 |
| V1.1 | MACD、RSI | 最常用副图指标，验证多线和柱状输出。 |
| V1.2 | Bollinger Bands、ATR | 引入波动率计算和通道展示。 |
| V1.3 | VWAP、OBV | 引入成交量加权和量价累积。 |
| V1.4 | Supertrend、ADX | 引入递推状态类指标和趋势强度判断。 |

## 10. 计算实现注意事项

1. 所有输入数据必须按时间升序计算，不能倒序计算后再反转。
2. 指标计算应保留足够历史 K 线，页面显示 200 根 K 线时，后端最好额外加载指标预热数据。
3. 递推类指标如 EMA、RSI、ATR、ADX、Supertrend 会受到初始化方式影响，必须固定规则。
4. 计算时使用 Decimal 或统一精度浮点；展示时再格式化小数位。
5. 空值、缺 K、成交量为 0、`high = low` 等边界情况要统一处理。
6. 数据源口径要写入元信息，例如交易所、市场类型、周期、价格源、参数、计算版本。

## 11. 推荐指标元信息结构

```json
{
  "name": "rsi",
  "display_name": "RSI",
  "category": "momentum",
  "overlay": false,
  "inputs": [
    {
      "name": "period",
      "type": "integer",
      "default": 14,
      "min": 1
    },
    {
      "name": "source",
      "type": "source",
      "default": "close"
    }
  ],
  "outputs": [
    {
      "name": "rsi",
      "type": "line",
      "color": "#7c3aed"
    }
  ]
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| name | 程序内部指标名。 |
| display_name | 前端显示名。 |
| category | 指标分类，例如 `trend`、`momentum`、`volume`、`volatility`。 |
| overlay | 是否叠加在主 K 线图上。 |
| inputs | 用户可配置参数。 |
| outputs | 指标输出线、柱、区域的定义。 |

这套元信息可以让前端自动生成参数面板和图层样式，后端只负责按参数计算数据。
