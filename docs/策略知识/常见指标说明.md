## 1. 使用说明

TradingView 的指标分为内置指标、社区脚本和用户自定义 Pine Script。本文只整理交易中最常见、也最适合项目优先支持的一组技术指标。

技术指标的本质是把历史价格、成交量、波动率或盘口衍生数据转换成更容易观察的信号。它们不能保证预测未来，使用时应结合趋势、成交量、市场环境和风险控制。

## 2. 趋势类指标

| 指标 | 作用 | 简要算法 |
| --- | --- | --- |
| MA / SMA 简单移动平均线 | 平滑价格，判断趋势方向、动态支撑压力和均线排列。 | 最近 N 根 K 线收盘价求平均：`SMA = sum(close, N) / N`。 |
| EMA 指数移动平均线 | 比 SMA 更重视近期价格，适合短线趋势跟踪。 | `alpha = 2 / (N + 1)`，`EMA = close * alpha + EMA_prev * (1 - alpha)`。 |
| MACD | 判断趋势方向、趋势动量、金叉死叉和背离。 | 常用 12/26/9：`DIF = EMA12 - EMA26`，`Signal = EMA(DIF, 9)`，`Histogram = DIF - Signal`。 |
| Supertrend | 用趋势线提示多空状态，也常用于移动止损。 | 基于 ATR 计算上下轨；价格突破上轨转多，跌破下轨转空。 |
| Ichimoku Cloud 一目均衡表 | 同时观察趋势、支撑压力、动量和未来云区。 | 由转换线、基准线、先行 Span A/B、迟行线组成，多数线用指定周期高低价中点计算。 |
| Parabolic SAR | 跟踪趋势反转点，常用于止损或离场。 | 根据极值点 EP 和加速因子 AF 递推 SAR，趋势延续时 AF 逐步增加。 |
| ADX / DMI | 衡量趋势强度，并区分正向/负向趋势力量。 | 先算 +DM、-DM 和 TR，再平滑为 +DI、-DI，ADX 是 DX 的平滑结果。 |

## 3. 动量与震荡类指标

| 指标 | 作用 | 简要算法 |
| --- | --- | --- |
| RSI 相对强弱指数 | 判断超买超卖、动量变化和顶底背离。 | 常用 14 周期：`RS = 平均上涨 / 平均下跌`，`RSI = 100 - 100 / (1 + RS)`。 |
| Stochastic 随机指标 | 判断收盘价在近期高低区间中的位置，适合震荡行情。 | `%K = (close - lowestLow) / (highestHigh - lowestLow) * 100`，`%D` 是 `%K` 的移动平均。 |
| Stochastic RSI | 把 Stochastic 算法应用到 RSI 上，信号更灵敏。 | `StochRSI = (RSI - min(RSI, N)) / (max(RSI, N) - min(RSI, N))`。 |
| CCI 商品通道指数 | 衡量价格偏离统计均值的程度，寻找周期性拐点。 | `TP = (high + low + close) / 3`，`CCI = (TP - SMA(TP)) / (0.015 * MeanDeviation)`。 |
| Momentum 动量 | 观察当前价格相对过去价格的变化幅度。 | `Momentum = close_today - close_N_periods_ago`，也可用比值形式。 |
| ROC 变化率 | 用百分比表示价格变化速度。 | `ROC = (close - close_N_periods_ago) / close_N_periods_ago * 100`。 |
| Williams %R | 判断收盘价接近近期高点还是低点，常用于超买超卖。 | `%R = (highestHigh - close) / (highestHigh - lowestLow) * -100`。 |

## 4. 成交量与量价类指标

| 指标 | 作用 | 简要算法 |
| --- | --- | --- |
| Volume 成交量 | 判断市场参与度、突破可信度和流动性。 | 直接统计当前 K 线成交量，可叠加成交量均线。 |
| Volume MA 成交量均线 | 平滑成交量，识别放量或缩量。 | 对最近 N 根成交量求 SMA 或 EMA。 |
| VWAP 成交量加权平均价 | 观察日内或锚定区间的平均成交成本。 | `VWAP = sum(price * volume) / sum(volume)`，常用典型价作为 price。 |
| Anchored VWAP 锚定 VWAP | 从指定时间点开始计算平均成交成本。 | 与 VWAP 相同，但累计区间从用户选择的锚点开始。 |
| OBV 能量潮 | 用成交量累积确认趋势或发现量价背离。 | 收盘价上涨则加当期成交量，下跌则减当期成交量，持平不变。 |
| MFI 资金流量指标 | 类似带成交量权重的 RSI，用于观察资金流入流出。 | 先算典型价和资金流，区分正/负资金流，再按 RSI 类似公式归一化到 0-100。 |
| CMF 蔡金资金流 | 衡量一段时间内收盘位置与成交量共同反映的资金压力。 | 先算 Money Flow Multiplier，再乘成交量，最后对 N 周期求和并除以成交量总和。 |
| Accumulation/Distribution 累积/派发线 | 判断资金累积或派发趋势。 | 用收盘价在高低价区间的位置给成交量加权，并累加成线。 |

## 5. 波动率类指标

| 指标 | 作用 | 简要算法 |
| --- | --- | --- |
| ATR 平均真实波幅 | 衡量波动大小，常用于止损距离、仓位控制和过滤低波动行情。 | `TR = max(high-low, abs(high-prevClose), abs(low-prevClose))`，ATR 是 TR 的平滑均值。 |
| Bollinger Bands 布林带 | 观察价格相对均值的偏离、波动收缩和突破。 | 中轨通常为 SMA，`上轨 = SMA + K * 标准差`，`下轨 = SMA - K * 标准差`。 |
| Bollinger Bands Width 布林带宽度 | 衡量布林带收缩或扩张，寻找低波动后的突破机会。 | 常见算法：`(上轨 - 下轨) / 中轨`。 |
| Keltner Channels 肯特纳通道 | 用 ATR 构建趋势通道，比布林带更偏波动跟踪。 | 中轨常用 EMA，`上轨 = EMA + K * ATR`，`下轨 = EMA - K * ATR`。 |
| Historical Volatility 历史波动率 | 观察资产过去收益率的波动强度。 | 对收益率序列求标准差，通常再做年化处理。 |
| Standard Deviation 标准差 | 衡量价格围绕均值的离散程度。 | 对 N 周期价格与均值的偏差平方求平均后开方。 |

## 6. 支撑压力与结构类指标

| 指标 | 作用 | 简要算法 |
| --- | --- | --- |
| Pivot Points 枢轴点 | 根据上一周期高低收快速计算支撑压力位。 | `P = (High + Low + Close) / 3`，再派生 R1/R2/S1/S2 等价位。 |
| Fibonacci Retracement 斐波那契回撤 | 标记趋势回调中的潜在支撑压力。 | 在一段高低点之间按 23.6%、38.2%、50%、61.8%、78.6% 等比例划分。 |
| Donchian Channels 唐奇安通道 | 观察 N 周期最高价和最低价形成的突破区间。 | 上轨为 N 周期最高价，下轨为 N 周期最低价，中轨常为两者平均。 |
| Support / Resistance 支撑压力 | 标记历史多次反应的价格区域。 | 常见做法是识别摆动高低点、成交密集区或多次触碰未突破区域。 |
| Market Structure 市场结构 | 判断高高、低低、突破结构和趋势切换。 | 通过 swing high / swing low、BOS、CHOCH 等结构规则识别。 |
| Order Block 订单块 | 尝试标记趋势启动前的关键供需 K 线区域。 | 通常寻找突破前的最后一段反向 K 线或盘整区，再向右延伸为观察区。 |
| Fair Value Gap / FVG | 标记快速波动留下的价格不连续区域。 | 常见三根 K 线算法：第一根高点低于第三根低点为上行缺口，反之为下行缺口。 |

## 7. 加密货币常见衍生指标

| 指标 | 作用 | 简要算法 |
| --- | --- | --- |
| Funding Rate 资金费率 | 衡量永续合约多空支付方向，辅助判断拥挤程度。 | 交易所按规则周期性计算，多数基于永续价格与现货/指数价格偏离。 |
| Open Interest 持仓量 | 观察未平仓合约规模，判断杠杆资金是否增加。 | 所有未平仓合约数量或名义价值之和。 |
| Long / Short Ratio 多空比 | 观察账户数或持仓量层面的多空倾斜。 | 多头账户/仓位数量除以空头账户/仓位数量，具体口径取决于交易所。 |
| CVD 累计成交量差 | 判断主动买卖力量的累积变化。 | 将主动买入成交量减主动卖出成交量，并随时间累加。 |
| Liquidation Levels 爆仓价位 | 观察高杠杆仓位可能被强平的价格区域。 | 根据合约仓位、杠杆、保证金和交易所强平规则估算，公开工具多为近似模型。 |
| Fear & Greed 恐惧贪婪指数 | 观察市场情绪温度，适合宏观过滤。 | 通常综合波动率、动量、社交热度、搜索趋势等数据，不同来源口径不同。 |

## 8. 项目优先级建议

V1 最建议优先支持：

| 优先级 | 指标 |
| --- | --- |
| 必做 | SMA / EMA、MACD、RSI、Bollinger Bands、Volume、Volume MA |
| 建议做 | ATR、OBV、VWAP、Supertrend、ADX / DMI |
| 可延后 | Stochastic、CCI、MFI、CMF、Pivot Points、Donchian Channels |
| 视数据源决定 | Funding Rate、Open Interest、Long / Short Ratio、CVD、Liquidation Levels |

## 9. 实现注意事项

1. 同名指标在不同平台、不同脚本中可能存在参数差异，例如周期、平滑方式、价格源和是否忽略空值。
2. TradingView 内置指标可以作为视觉和参数参考，但项目实现时应明确每个指标的计算口径。
3. 加密货币衍生指标依赖交易所数据源，不同交易所口径可能不一致。
4. 指标信号应尽量输出结构化字段，例如当前值、方向、阈值状态、最近交叉、背离候选，而不是只输出一句自然语言判断。
