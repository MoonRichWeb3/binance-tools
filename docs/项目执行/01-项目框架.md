# 01 — 项目框架

> 本文描述当前仓库 **binance-tools** 的总体结构与约定，随代码演进可修订。

## 1. 目标与边界

- **定位**：围绕币安开放平台能力构建的 **Rust** 工具与可复用模块（公共逻辑在 `src/`；演示应用在 `examples/` 下的**独立子工程**，按需新增）。
- **文档**：技术资料与 API 摘录放在 [`../知识库/`](../知识库/README.md)；**执行与计划** 放在本目录 `docs/项目执行/`。

## 2. 技术栈

| 项 | 说明 |
| --- | --- |
| 语言 | Rust（`edition = "2024"`，见根目录 `Cargo.toml`） |
| 包名 | `binance_tools`：同时提供 **库**（`src/lib.rs`）与默认 **二进制**（`src/main.rs`） |
| Binance SDK | 根 crate 接入官方 `binance-sdk`，启用 `spot` 与 `rustls-tls`，统一从 `src/binance/` 暴露配置与后续业务封装 |
| 桌面 UI | `examples/desktop-gpui/` 接入 `gpui-component`，固定使用 `v0.5.1` tag；主题通过 `ThemeRegistry` 加载 `themes/*.json`；作为 workspace 成员通过路径依赖复用根 crate |
| Windows 图标 | `examples/desktop-gpui/build.rs` 使用 `winresource` 把 `assets/app.ico` 嵌入 exe |

## 3. 源码与示例约定

| 路径 | 用途 |
| --- | --- |
| **`src/lib.rs`** | 库入口，统一导出 `app`、`ai`、`binance`、`db`、`square` 等公共模块 |
| **`src/app/`** | 应用级常量、能力枚举与跨入口共享状态 |
| **`src/ai/`** | AI 配置、Provider 选择、OpenAI-compatible 流式请求和错误格式化 |
| **`src/binance/`** | Binance SDK 集成入口、生产/测试网配置、现货和 Web market product 数据封装 |
| **`src/db/`** | SQLite 建表、缓存读写、市场榜单、现货和币安广场数据访问 |
| **`src/square/`** | 币安广场发送执行器、定时任务渲染和返回码处理 |
| **`src/main.rs`** | 默认命令行入口（薄封装，复杂逻辑放在库模块） |
| **`examples/desktop-gpui/`** | GPUI Component 桌面示例工程；workspace 包名为 `desktop-gpui`，通过 `binance_tools = { path = "../.." }` 复用根 crate；内部按 `app.rs`、`theme.rs`、`ui/` 分层 |
| **`docs/知识库/`** | 第三方官方文档摘录、链接索引、离线 `llms-full` 等参考资料 |
| **`docs/项目执行/`** | 本项目规划、框架、发布步骤、排障记录 |

根目录常用命令：

```text
cargo build              # 构建库 + 默认二进制
cargo run                # 运行默认二进制
cargo test               # 运行根 crate 单元测试
```

桌面示例：

```text
cargo run -p desktop-gpui
```

说明详见 [`../../examples/README.md`](../../examples/README.md)。

桌面 Release 构建：

```text
cargo build -p desktop-gpui --release
```

生成的 Windows 可执行文件位于 `target/release/desktop-gpui.exe`。当前 exe 图标来自 `examples/desktop-gpui/assets/app.ico`，由 `build.rs` 在 Windows 编译时嵌入。

## 5. 集成边界

| 能力 | 当前落点 | 后续扩展 |
| --- | --- | --- |
| Binance 现货 | `src/binance/spot.rs` 和 `src/db/spot.rs`，支持币种缓存、日 K 缓存、日均线信号和 K 线图数据 | 后续增加筛选、详情、账户和交易能力 |
| Binance Web market | `src/binance/market.rs` 和 `src/db/market.rs`，用于 Dashboard 市场榜单和 AI 分析 | 后续可增加更多市场统计聚合 |
| 币安广场 | `src/square/mod.rs` 和 `src/db/square.rs`，支持 Key、任务、日志和定时发送 | 后续可接入 AI 内容生成工作流 |
| AI | `src/ai/mod.rs` 和桌面 `ui/ai/`，支持 Provider 配置、模型选择和 OpenAI-compatible 流式输出 | 后续补齐 Anthropic/Google 流式、工具调用和 MCP |
| CLI | `src/main.rs` 只做薄入口，打印当前默认配置 | 后续改为参数解析、命令分发和脚本化操作 |
| 桌面 UI | `examples/desktop-gpui/src/` 使用 GPUI Component 渲染桌面壳，`theme.rs` 集中处理主题注册与切换 | 后续页面只调用根 crate 业务接口，不直接散落 SDK 调用 |

## 7. 后续扩展建议

- **Binance 子模块**：现有 `market.rs`、`spot.rs` 继续保持业务封装，后续再扩展 `account.rs`、`trade.rs` 等。
- **示例子工程**：继续在 `examples/<名称>/` 下维护独立 Cargo 工程；需要从根目录用 `-p` 运行的示例加入 `[workspace].members`。
- **桌面 UI 分层**：`main.rs` 保持薄入口；启动和窗口放 `app.rs`；主题加载、主题名和切换 API 放 `theme.rs`；窗体栏控件放 `ui/title_bar.rs`；具体页面放 `ui/`，后续可扩展 `ui/market.rs`、`ui/account.rs`、`ui/trade.rs`。
- **配置与密钥**：AI provider 可保存到 `config/ai.json`；交易类 Key 后续应使用环境变量或本地忽略配置。
- **测试策略**：纯配置与签名等确定性逻辑放单元测试；真实 Binance 请求默认走测试网，并用显式开关控制。
