# 01 — 项目框架

> 本文描述当前仓库 **binance-tools** 的总体结构与约定，随代码演进可修订。

## 1. 目标与边界

- **定位**：围绕币安开放平台能力构建的 **Rust** 工具与可复用模块（公共逻辑在 `src/`；演示应用在 `examples/` 下的**独立子工程**，按需新增）。
- **文档**：技术资料与 API 摘录放在 [`../知识库/`](../知识库/README.md)；**执行与计划** 放在本目录 `docs/项目执行/`。

## 2. 技术栈

| 项 | 说明 |
| --- | --- |
| 语言 | Rust（`edition = "2024"`，见根目录 `Cargo.toml`） |
| 包名 | `binance_tools`：同时提供 **库**（`src/lib.rs`）与默认 **二进制**（`src/main.rs`） |
| 依赖 | 按功能在 `Cargo.toml` 的 `[dependencies]` 中声明；官方 HTTP 侧可优先参考 [`../知识库/币安sdk/README.md`](../知识库/币安sdk/README.md) 中的 `binance-sdk` |

## 3. 源码与示例约定

| 路径 | 用途 |
| --- | --- |
| **`src/lib.rs` + `src/**/*.rs`** | **公共代码**：库入口与子模块；单元测试与 `tests/` 针对库行为编写 |
| **`src/main.rs`** | 默认命令行入口（薄封装，复杂逻辑放在库模块） |
| **`examples/`** | **示例 Rust 工程**：每个示例子目录为**独立 Cargo 项目**（自带 `Cargo.toml`），通过路径依赖引用根 crate；**不使用**根包下的 `examples/*.rs` 单文件示例形态 |

根目录常用命令：

```text
cargo build              # 构建库 + 默认二进制
cargo run                # 运行默认二进制
```

示例子工程创建后，在其目录内执行 `cargo run`（若采用 workspace，也可用 `cargo run -p <包名>`，视后续 `Cargo.toml` 配置而定）。

说明详见 [`../../examples/README.md`](../../examples/README.md)。

## 5. 文档分层约定

| 目录 | 用途 |
| --- | --- |
| `docs/知识库/` | 第三方官方文档摘录、链接索引、离线 `llms-full` 等**参考资料** |
| `docs/项目执行/` | **本项目** 的规划、框架、发布步骤、排障记录 |

## 6. 后续扩展建议

- **子模块**：在 `src/` 下新增 `mod xxx` 或目录 `src/xxx/mod.rs`，由 `lib.rs` 统一 `pub mod` 控制对外暴露范围。
- **示例子工程**：在 `examples/<名称>/` 下 `cargo new` 或手写 Cargo 工程；需要时在仓库根 `Cargo.toml` 配置 `[workspace]` 成员以便一键构建。
- **配置与密钥**：API Key 等不放仓库；使用环境变量或本地配置文件（并在 `.gitignore` 中忽略）。
- **集成测试**：库行为可放在 `tests/*.rs`。
