# 04 币安广场

本文记录当前 `desktop-gpui` 中币安广场功能的实现范围。

## 功能定位

币安广场模块用于：

- 保存币安广场 OpenAPI Key。
- 立即发送一条输入内容。
- 按预计发送时间保存发送任务。
- 开启 AI 分析后，每 1 小时生成一条广场发送草稿任务。
- 手动执行到期任务。
- 启动/停止页面内调度器。
- 选中任务后立即发送，并把任务状态同步更新到表格。
- 查看发送日志。
- 删除发送日志。

## 发送接口

币安广场消息通过以下接口发送：

```text
POST https://www.binance.com/bapi/composite/v1/public/pgc/openApi/content/add
```

请求头：

| Header | 说明 |
| --- | --- |
| `X-Square-OpenAPI-Key` | 币安广场 API Key |
| `Content-Type: application/json` | JSON 请求体 |
| `clienttype: binanceSkill` | Binance OpenAPI 要求的客户端类型 |

请求体：

```json
{
  "bodyTextOnly": "最终发送正文"
}
```

业务执行逻辑位于：

```text
src/square/mod.rs
```

## 任务页面

`SquareTasksPage` 当前布局：

- 顶部是币安广场任务概览卡片，支持折叠；卡片内展示 AI 下次分析倒计时和发送调度倒计时。
- 概览卡右侧有两个开关：`AI 草稿` 和 `发送调度`。
- 中部是任务编辑卡片，支持折叠；选中表格任务后会回填正文、任务名称和预计发送时间。
- 编辑区的预计发送时间支持快捷按钮：`现在`、`+30 分钟`、`+1 小时`、`明天 09:00`。
- 下方是任务表格，展示 `ID`、标题、来源、内容、预计发送时间、状态、更新时间。

主要按钮与状态：

| 操作 | 说明 |
| --- | --- |
| `AI 草稿` | 开启后，每 1 小时生成一条 AI 市场分析草稿任务，默认 `draft`，不会直接发送 |
| `立即生成` | 立即执行一次 AI 分析；执行中按钮显示 loading，完成后写入任务表并刷新 |
| `发送调度` | 开启后，每 30 分钟检查一次到期 `pending` 任务 |
| `发送到期` | 手动执行当前已到期的 `pending` 任务 |
| `立即发送` | 发送编辑区当前正文；如果当前选中了任务，会把该任务状态同步改为 `sent`、`skipped` 或 `failed` |
| `新增任务` | 把编辑区内容保存为手动任务，默认可进入发送流程 |
| `保存修改` | 更新当前选中任务的标题、名称、正文和预计发送时间 |
| `修改状态` | 下拉选择任务状态，可切换 `draft`、`pending`、`sent`、`skipped`、`failed` |
| `确认发送` | 把当前选中任务改为 `pending`，之后才会被调度器处理 |
| `删除` | 删除当前选中任务；需要再次确认，删除后清空编辑区 |

任务只在 `send_status = pending` 且 `scheduled_at <= 当前本地时间` 时发送一次。调度器先把任务原子标记为 `sending`，避免重复发送；发送后更新为 `sent`、`skipped` 或 `failed`。

## AI 分析生成任务

AI 分析逻辑位于：

```text
src/square/mod.rs
```

执行规则：

- [✓] 【完成】 默认分析 `USDT` 市场。
- [✓] 【完成】 先查询当天所有 AI 任务标题，例如 `$RONIN`、`$AI`。
- [✓] 【完成】 从本次市场数据中排除当天已经使用过的币种。
- [✓] 【完成】 取剩余数据前 50 条发送给当前 AI 模型。
- [✓] 【完成】 使用 `config/ai.json` 中的 `agent.default_model`；右侧 AI Chat 切换模型时会同步更新这个默认模型。
- [✓] 【完成】 AI 输出必须是一行 `$币种 极短理由`。
- [✓] 【完成】 中文汉字数量必须在 40 到 90 个之间，不包含 `$币种代码`、空格、标点和英文字母。
- [✓] 【完成】 输出不能包含涨幅、价格、百分比、目标位或交易诱导词。
- [✓] 【完成】 文案会避免重复套用“值得跟踪/值得追踪/值得关注”等固定结尾；保存前也会做一次常见重复短语清洗。
- [✓] 【完成】 生成成功后写入 `binance_square_tasks`，`source_type = ai_market_analysis`，`send_status = draft`。
- [✓] 【完成】 `draft` 只作为数据入口和人工确认结果，不会被发送调度器直接发布到币安广场。
- [✓] 【完成】 用户需要在任务表格中选中草稿，必要时修改正文和预计发送时间，然后点击 `确认发送`，任务才会进入 `pending`。
- [✓] 【完成】 如果 AI 有返回但格式不完全合规，也会写入 `draft` 草稿，并在 `binance_square_ai_logs` 里记录 `draft_invalid`，便于人工修改。
- [✓] 【完成】 币安广场 AI 规则和兜底文案从 `ai_rules` 表读取，不再在业务流程里写死。

AI 生成任务每日最多 100 条，避免调度异常导致刷屏。单次生成最多尝试 3 次，每次失败后间隔 5 分钟再执行下一次。

## 发送日志

发送日志写入 `binance_square_send_logs`。

日志页面当前能力：

- 从 SQLite 查询最近最多 `1000` 条发送记录。
- 表格展示 `ID`、`Task`、`Status`、`Code`、`Retry`、`Sent At`、`Message`、`Error`。
- 支持表格排序。
- 支持复制选中单元格、行或列。
- 支持手动刷新。
- 支持删除选中日志，删除前需要二次确认。

## 返回码处理

| 场景 | 返回码 / 特征 | 当前策略 |
| --- | --- | --- |
| 敏感词 | `20002`、`20022` | 记录失败原因，跳过当前正文 |
| 每日限额 | `220009` | 停止后续发送 |
| Key 过期 | `220004` | 停止后续发送并提示重新设置 Key |
| 重复发布 | duplicate/already 等提示 | 静默跳过，更新任务状态 |
| 网络错误 | 连接失败、超时、临时 5xx | 最多重试 3 次 |
| 成功 | 服务端成功响应 | 写日志并更新任务状态 |

网络错误重试使用指数退避：`1s`、`2s`、`4s`。

## UI 约定

- 任务表格和日志表格使用统一 `DataTable` 风格。
- 表格区域随窗口高度伸缩。
- 按钮统一使用 `xsmall()`。
- 状态信息在页面内展示，不弹系统窗口。
- 任务页和编辑区使用折叠按钮减少占屏，表格保持为主要工作区。
