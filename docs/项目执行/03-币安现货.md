# 03 — 币安现货

> 本文说明当前项目中的币安现货查询能力、UI 展示方式和后续扩展计划。

## 1. 功能定位

现货模块用于查询 Binance Spot 的公开市场基础信息，优先服务 `desktop-gpui` 的现货页面。接口代码放在根 crate 的 `src/binance/` 下，UI 只负责触发查询和展示结果。

| 项 | 说明 |
| --- | --- |
| 业务模块 | `src/binance/spot.rs` |
| UI 页面 | `examples/desktop-gpui/src/ui/spot.rs` |
| 查询接口 | Binance Spot REST `exchange_info` |
| 展示组件 | gpui-component `DataTable` |
| 本地数据库 | SQLite，默认文件 `db/binance_tools.sqlite` |
| 默认环境 | `BinanceSettings::production()`，现货页面直接请求 Binance 正式网 |

## 2. 当前能力

`src/binance/spot.rs` 提供：

- `SpotSymbolInfo`：项目内部使用的现货交易对信息结构。
- `fetch_spot_symbols(settings)`：异步查询全部现货交易对。
- `fetch_spot_symbols_blocking(settings)`：为桌面 UI 背景任务提供的同步包装，内部创建 Tokio runtime。
- `DailyMaSignal`：USDT 现货日均线信号结构，包含当前价、均线价和偏离百分比。
- `fetch_usdt_daily_ma_signals(settings, days)`：查询所有 `USDT` 现货交易对，按日 K 收盘价计算指定天数均价。
- `fetch_usdt_daily_ma_signals_blocking(settings, days)`：桌面 UI 使用的同步包装。

`src/db/spot.rs` 提供：

- `load_or_fetch_spot_symbols_blocking(settings)`：优先从 SQLite 查询现货币种；数据库为空时请求 Binance 并写入 SQLite。
- `refresh_spot_symbols_blocking(settings)`：强制请求 Binance 并替换 SQLite 中的现货币种数据。
- `list_spot_symbols(connection)`：从 `spot_symbols` 表查询全部币种。
- `count_distinct_base_assets(connection)`：通过 SQL 统计 `base_asset` 去重数量。
- `load_cached_usdt_daily_ma_signals_blocking(days)`：只从 SQLite 查询已满足指定天数日 K 缓存的 USDT 均线信号。
- `load_or_fetch_usdt_daily_ma_signals_blocking(settings, days)`：先读缓存，再为缺失币种请求 Binance 日 K 并写入 `spot_klines_1d`。
- `list_spot_daily_klines_blocking(symbol, limit)`：从 SQLite 查询指定交易对最近的日 K 数据，供 Candlestick 页面展示。

当前展示字段：

| 字段 | 来源 | 说明 |
| --- | --- | --- |
| `symbol` | `exchangeInfo.symbols[].symbol` | 交易对，例如 `BTCUSDT` |
| `status` | `status` | 交易状态 |
| `base_asset` | `baseAsset` | 基础资产 |
| `quote_asset` | `quoteAsset` | 计价资产 |
| `base_asset_precision` | `baseAssetPrecision` | 基础资产精度 |
| `quote_asset_precision` | `quoteAssetPrecision` | 计价资产精度 |
| `order_types` | `orderTypes` | 支持订单类型 |
| `spot_trading_allowed` | `isSpotTradingAllowed` | 是否允许现货交易 |
| `margin_trading_allowed` | `isMarginTradingAllowed` | 是否允许杠杆交易 |

## 3. UI 设计

`desktop-gpui` 当前在主窗口中加入 `现货` 页面区域：

- `现货` 入口位于窗体栏，和 `Theme` 放在同一组。
- 点击 `现货` 后出现下拉菜单，当前菜单项为 `币种`。
- 点击 `币种` 后，主内容区整体切换为现货页面，只保留现货表格板块。
- 点击 `热力图` 后，主内容区切换为市场热力图页面。
- 点击 `日均线信号` 后，主内容区切换到日均线信号页面。
- 未点击 `现货` 时不创建现货表格页面。
- 现货页面首次打开时创建页面、触发查询并显示表格。
- 页面标题为 `现货`。
- 显示当前 Binance 环境、已加载交易对数量和 SQL 统计出的现货总数。
- 提供 `查询现货` 按钮，可手动刷新。
- [✓] 【完成】 现货币种页支持搜索 Symbol、Base、Quote。
- [✓] 【完成】 现货币种页支持按 `USDT`、`USDC`、`FDUSD`、`BTC`、`BNB`、`ETH`、`TRY`、`EUR` 切换计价资产。
- [✓] 【完成】 现货币种页按当前计价资产和搜索结果统计上涨、下跌、持平数量；涨跌数据来自 Binance Web product 行情缓存。
- 现货页面创建后自动查询一次，查询顺序为先读 SQLite，空库时再请求 Binance 正式网并落库。
- 查询中由 `DataTable` 显示加载态。
- 查询失败时在表格上方显示错误信息。
- 交易对信息放在 `DataTable` 中，支持滚动、列宽调整和基础排序。

### 市场热力图页面

- 页面入口位于 `现货` 下拉菜单，菜单项为 `热力图`。
- 页面文件为 `examples/desktop-gpui/src/ui/heatmap.rs`。
- 数据来源为 `binance_market_products_cache`，和市场榜单页面共用 Binance Web product 缓存。
- 默认展示 `USDT` 交易对，支持切换所有缓存中存在的计价资产。
- 支持搜索交易对、基础资产、名称、市场分区和标签。
- 方块颜色表示 24 小时涨跌：上涨为绿色，下跌为红色，持平为灰色；使用接近移动端热力图的低饱和色块。
- 方块大小按当前筛选结果中的权重占比分级，权重可在 `市值` 和 `成交额` 间切换；市值模式展示的是该币种市值占当前筛选总市值的比例，头部币种使用更大的 TradingView 风格主块。
- 热力图块使用显式分行布局，每行高度统一并自动铺满可用宽度，避免右侧留白和高低块造成的空洞。
- 默认最多展示前 100 个交易对，并隐藏当前筛选权重占比低于 0.25% 的小市值或低成交额交易对，避免页面过密。
- 大中块展示交易对、最新价、24h 涨跌幅和权重占比；小块仅展示基础资产和涨跌幅，避免文字拥挤。
- 点击任意方块进入该交易对的 1 日 K 线图，K 线图会自动补齐最新 120 日日线。
- 页面顶部展示上涨、下跌、持平数量，并提供同步和刷新按钮。

### 日均线信号页面

- 页面入口位于 `现货` 下拉菜单，菜单项为 `日均线信号`。
- 页面提供均线天数输入框，默认值为 `120`，范围限制为 `1` 到 `1000`。
- 点击 `日均线信号` 进入页面时，先加载数据库中已经满足指定天数日 K 缓存的币种。
- 点击 `查询信号` 后，保留已加载结果，并为缺失缓存的币种请求 Binance 正式网。
- 查询范围为状态 `TRADING`、允许现货交易、计价资产为 `USDT` 的交易对。
- 当前价使用缓存窗口内最新一根日 K 的收盘价。
- 均线价通过 `spot_klines` 中指定日期窗口内的日 K 收盘价 SQL 计算。
- 为避免触发 Binance 接口限流，每个 K 线请求后固定等待 `150ms`。
- 表格展示 `Symbol`、`Base`、当前价、均线价、偏离百分比、天数和样本数。
- 表格最后一列提供 `K` 图标按钮，点击后进入当前 USDT 交易对的 Candlestick 页面。
- 默认按偏离百分比从高到低排序，表格列支持再次排序。

### Candlestick 页面

- 页面文件为 `examples/desktop-gpui/src/ui/kline.rs`。
- 页面由日均线信号表格最后一列的 `K` 图标进入，不放在主菜单中单独暴露。
- 当前展示数据来自 SQLite `spot_klines` 表，默认读取所选 `symbol` 最近 `120` 根 `1d` K 线。
- 图表使用 gpui-component `CandlestickChart`，通过 `open_price`、`high_price`、`low_price`、`close_price` 绘制 Candlestick。
- 图表使用接近 Binance 行情页的深色盘面样式，包含顶部 OHLC、涨跌幅、成交量和 120 日高低点信息。
- 图表右侧显示 Y 轴价格，包含当前 120 根 K 线的最高价、中间价和最低价。
- 图表区域内标注当前 120 根 K 线的最高价和最低价。
- 图表下方增加成交量区域，使用 gpui-component `BarChart` 按 K 线涨跌显示成交量柱。
- 图表支持 `+` / `-` 按钮缩放，也支持在价格图区域滚轮缩放，缩放会改变当前可见 K 线根数。
- 鼠标移动到价格图区域时显示十字线，顶部时间、OHLC、涨跌幅和成交量会切换为鼠标对应日期的数据。
- 图表支持按住鼠标左键横向拖拽平移可见 K 线窗口，交互行为按 TradingView 方向设计。
- 十字线底部显示日期标签，右侧显示鼠标所在位置对应的价格标签。
- 价格图区域叠加 MA7、MA25、MA99 均线，顶部指标栏跟随鼠标显示对应日期的均线值。
- 阳线使用主题中的 `bullish` 颜色，阴线使用主题中的 `bearish` 颜色。
- 页面顶部展示当前交易对、已加载 K 线数量和最新收盘价，并提供 `刷新` 按钮重新读取数据库。
- 如果数据库中还没有该交易对的 K 线数据，页面提示先回到日均线信号页面执行查询，让缺失数据写入 `spot_klines`。

## 4. 分层约定

| 层级 | 职责 |
| --- | --- |
| `src/binance/spot.rs` | 调用 Binance SDK、处理 `exchange_info` 响应、转换为项目内部结构 |
| `src/db/spot.rs` | SQLite 建表后的现货币种和 K 线读写、空库回源 Binance、SQL 统计 |
| `examples/desktop-gpui/src/ui/spot.rs` | 现货页面状态、查询按钮、DataTable delegate、错误展示 |
| `examples/desktop-gpui/src/ui/heatmap.rs` | 市场热力图页面，按涨跌幅和权重展示交易对方块 |
| `examples/desktop-gpui/src/ui/ma_signal.rs` | 日均线信号页面、均线天数输入、DataTable delegate、错误展示 |
| `examples/desktop-gpui/src/ui/kline.rs` | 单交易对 Candlestick 页面，从 `spot_klines` 读取并绘制 K 线 |
| `examples/desktop-gpui/src/ui/dashboard.rs` | 将现货页面组合进主窗口 |

UI 层不直接调用 Binance SDK 的模型字段；需要展示的数据先转换为 `SpotSymbolInfo`。

## 5. 后续扩展

- 增加只看 `TRADING` 状态、只看现货允许交易等筛选项。
- 增加 24h ticker、最新价格、成交量等行情字段。

- 在根 crate 增加 `market.rs`、`account.rs`、`trade.rs` 后，再将 UI 页面拆成行情、账户、下单和订单列表。
